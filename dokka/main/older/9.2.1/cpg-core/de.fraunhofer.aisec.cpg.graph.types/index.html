HasSecondaryTypeEdge

fun interface HasSecondaryTypeEdge

The TypeResolver needs to be aware of all outgoing edges to types in order to merge equal types to the same node. For the primary type edge, this is achieved through the HasType interface. If a node has additional type edges (e.g. TypeParameterDeclaration.default) the node must implement the updateType method, so that the current type is always replaced with the merged one.
