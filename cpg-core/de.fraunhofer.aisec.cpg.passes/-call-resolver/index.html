Resolves CallExpression and NewExpression targets.

A CallExpression specifies the method that wants to be called via getName. The call target is a method of the same class the caller belongs to, so the name is resolved to the appropriate MethodDeclaration. This pass also takes into consideration that a method might not be present in the current class, but rather has its implementation in a superclass, and sets the pointer accordingly.

Constructor calls with ConstructExpression are resolved in such a way that their getInstantiates points to the correct RecordDeclaration. Additionally, the getConstructor is set to the according .

This pass should NOT use any DFG edges because they are computed / adjusted in a later stage.
